vue.js option api 生命鉤子

生命鉤子簡介

vue.js元件都有各自的生命週期，從建立到銷毀，不管是父元件還是子元件都有其生命週期。

而vue.js提供了Lifecycle Hooks，讓我們可以在元件不同生命週期操作相關程式碼。此處以option API為例。

圖片來源:重新認識 Vue.js | Kuro Hsu (1-7元件的生命週期與更新機制)

生命鉤子用途簡易說明

beforeCreate

  • 期間vue實例建立，但狀態事件尚未初始化，因此data() 和 computed還無法使用。這時候操作data是不會有反應的。

created

  • 已經可以用data(),computed, method, watch了，Server端API操作至少要在這個期間。但DOM還沒完成建立，因此還不能操作UI。

mounted

  • DOM創建完成，UI操作在這時候才會起作用。

圖例說明父元件、子元件間關係

初始渲染：

  • 首先父元件自己的beforeCreate 和 created。

  • 父元件開始掛載(beforeMount)，遇到子元件，子元件開始其生命週期(beforeCreate → created → beforeMount →mount)。

  • 子元件掛載完成後，父元件完成自己的掛載(mount)。

更新：

  • 資料變化觸發更新時，父元件先beforeUpdate

  • 接著子元件的beforeUpdate →updated

  • 最後父元件的updated

注意事項：

  • 只有子元件資料變化時，父元件不會變動。

  • 銷毀時則是先觸發父元件的beforeUnmount → ++子元件++的beforeUnmount→unmounted→最後是父元件的unmounted

利用v-if控制子元件渲染圖例

原則來說資料更新時beforeUpdate，不會觸發子元件beforeCreate ，因為在前面已經建立過了(beforeMount時遇到子元件會先建立子元件，最後才mounted)。

但如果用v-if控制子元件是否渲染時，父元件的beforeUpdate後會接著子元件的beforeCreate。(想像點擊按鈕之後才出現的一串文字，就是點擊按鈕後才被create)
