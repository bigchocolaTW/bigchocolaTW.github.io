Hexo建置及github action自動部屬

前言: 必備觀念與工具

這個網誌就是用Hexo (網站框架) 及github action自動部屬建置的，只需要push就能自動更新文章，免去每次都要重新轉靜態檔的流程:D

另外推推我使用的主題，使用手冊非常詳細好懂。 KEEP主題手冊

我會盡量簡單好上手，但如果你有以下技能，會更容易唷:

  1. Git / Github 基本概念

  2. npm 基本概念

  3. Markdown 語法概念

開始前請先安裝:

  1. Node.js

  2. Git

  3. VSCode(選配)

安裝過程也可以參考:

Hexo官方文檔

Easy Hexo

教得很詳細我就告辭了

安裝Hexo

以下我用VSCode內的git bash 操作示範:D

  1. 先在喜歡的路徑建立資料夾，並進入

  1. 右鍵選擇 【以Code開啟】

  1. 開啟Git Bash終端機

  2. 輸入相關指令

    此處的-g 為global，全域安裝的意思

    npm install -g hexo-cli

    安裝完成後輸入

    hexo init

    npm install

    hexo version

    hexo安裝完成~!

  3. 在本機執行看看

    hexo server

    大功告成

    剛開始而已啦XD 關掉案Ctrl + C

    若要打指令都要先關掉哦

    發布文章

    1. 輸入指令

      hexo new [layout] <title>

      EX. hexo new "my first post" 或 hexo new 我的第一篇文章

      layout這邊可以先空白，title就是你的文章標題。

      若標題有空白，要記得用引號包住哦~

      文章會出現在source底下的_posts

    2. 編輯md檔案

      tags 就是標籤

      categories 是分類(資料夾)，可以參考此文章做分類設定

      文章可以使用markdown語法編輯，網路上很多教學，可以找自己能吸收的~

      其實還有很多教學設定，可以多多參考官方文檔或其他教學，在此就不多做贅述了。

      強烈建議先進行基本設定，尤其是根目錄底下的_config.yml，預設url: http://example.com，記得改為你後續的網址。如果目前沒有，就先維持原狀吧~

    重要指令

    hexo generate 產生靜態文件(html/css/js等文件)，可在 pubilic 檔案夾內查閱

    hexo clean 清除快取、靜態文件(會移除pubilic 檔案夾，如果有更新檔案但沒跑出來，可以使用此指令，記得再重新$ hexo generate)

    hexo deploy 依_config.yml 內配置部屬到指定位置

    題外話，有些教學文章指令會寫 $hexo generate，是為了表示這是一段要命令的意思，實際上可以不用打上去唷。當然還是要看場合，因為也有時候是別的用途~

    部署到github上

    嗨~我先假設你已經有Github帳號~沒有的話請趕快註冊一個~

    接下來我也會盡量圖文講解，讓我們來試試看吧:D

    基本概念

    接下來我們要開兩個repository，你可以想像這是裝你的程式碼的倉庫。

    一個設定為私人，存放原始碼。一個設定公開，存放靜態文件。

    當然如果過程中覺得太困難，也未必要使用這種方式，只是這可以避免你的原始碼公開，又能保有自動部屬的便利性。

    太困難了話，可以先嘗試手動部署，慢慢熟練再來挑戰也沒問題的!

    1. 建立repository

      Repository name可以隨便你打，但要記得存放原始碼的在4.設定成Private。而存放靜態檔的設定為Public。

      記得存檔

      這裡要注意一下，如果你之後打算用github pages功能，Repository name 這邊請務必要填 [你的github帳號].github.io，可參考這篇教學

      EX. bigchocolaTW.github.io

      1. 檔案git到github上

        希望在這之前你能先有一些git/github觀念了，如果沒有，可以先看看這篇，或者就照抄作業囉XD

        git init 初始化工作目錄

        git add . 把工作目錄內所有檔案加入staging area

        git commit -am '<填寫版本資訊>' 將所有staging area內容(變更等等資訊)送進本地倉庫(loacl repository) 並加註版本資訊

        EX. git commit -am '我的第一版'

        接下來..

      2. 設定自動部屬

        接下來可以直接看這篇文章~因為他寫得非常詳細然後我也累了哈哈哈

        如何使用 GitHub Actions 自动部署 Hexo 博客

        但在這之前有幾件事情要注意

        文章中提到的hexo-deploy.yml 文件内容，請不要直接複製貼上，一定會出錯

        原因是這樣的，

        首先，現在github直接複製他的remote命令，預設分支會是main，可以參考這則新聞

        git branch -M main，這個main就是預設分支的名稱，

        而有些文章預設是master，所以這邊要記得修改為你的預設分支名稱。

        如果分支是main記得改main

        此外，此處記得修改為你的公開repository~

        記得改為你的repository名稱(公開的那個)

        如果你的公開repository有出現檔案，就代表大功告成囉~

        如果你是使用github pages 網址就會是 https://<你的github帳號>.github.io/

        像我就是https://bigchocolatw.github.io/

      結語

      謝謝你看到這邊，也辛苦你啦~希望有成功幫助你建立自己的網誌。

      這裡面也有很多設定要摸索，慢慢學習囉~我也加油你也加油

      很感謝網路上各路專家的無私分享，才終於成功讓我建立這個網誌^q^
